Atividade 03: CRUD de Dispositivos — Passo a Passo com fetch()

Programação III · Prof. Élder F. F. Bernardi · IFSul — Campus Passo Fundo

Trabalho em dupla · Prazo: 2 aulas · Valor: 1,0 ponto

Objetivos de Aprendizagem

Ao concluir esta atividade, você será capaz de:

  • Consumir uma API REST pública usando fetch() com async/await;
  • Implementar as quatro operações CRUD (Criar, Ler, Atualizar, Excluir) a partir do navegador;
  • Armazenar dados retornados pela API em vetores locais no JavaScript;
  • Manipular o DOM (consultar e criar elementos) para renderizar dados na tela;
  • Compreender o fluxo completo: ação do usuário → requisição HTTP → resposta → atualização do vetor local → re-renderização da tela.

Contexto

Nas aulas anteriores, vimos que o fetch() é a ferramenta nativa do JavaScript para fazer requisições HTTP a servidores remotos. Nesta atividade, vamos aplicar todos os verbos CRUD (GET, POST, PUT, DELETE) contra a API pública https://api.restful-api.dev/objects, que simula um banco de dados de dispositivos eletrônicos.

⚠️ Atenção: Esta API é pública e de testes. Ela não persiste os dados entre sessões. Isso significa que, se vocês cadastrarem um novo dispositivo, ele existirá apenas enquanto a aba do navegador estiver aberta. Ao recarregar a página, os dados voltam ao estado original da API. Isso é normal e esperado — o objetivo da atividade é praticar as requisições, não armazenar dados de verdade.

O fluxo de trabalho será:

Usuário clica num botão
    → JavaScript dispara fetch() para a API
        → API responde com JSON
            → JavaScript salva/atualiza o vetor local
                → Função renderizar() redesenha a tabela na tela

Vocês receberão um boilerplate (esqueleto) com todo o HTML, CSS e a estrutura do JavaScript já prontos. O trabalho de vocês é implementar as funções fetch(), seguindo o passo a passo abaixo.

Passo 1 — Listar todos os dispositivos (GET)

O que vamos fazer: Buscar todos os dispositivos da API, salvar no vetor local e renderizar a tabela.

Neste passo, vamos implementar duas funções: renderizar() e listarDispositivos().

1.1 — Implementar a função renderizar()

Esta é a função mais importante do projeto. Ela é responsável por ler o vetor dispositivos e criar as linhas da tabela no HTML. Será chamada após cada operação.

Substitua o conteúdo da função renderizar() pelo código abaixo:

function renderizar() {
  // 1. Limpar TODAS as linhas antigas da tabela
  //    (innerHTML = '' apaga todo o conteúdo interno do <tbody>)
  corpoTabela.innerHTML = '';

  // 2. Para cada item do vetor, criar uma linha <tr> na tabela
  for (let i = 0; i < dispositivos.length; i++) {
    const item = dispositivos[i];

    // 2a. Criar o elemento <tr> (linha da tabela)
    const linha = document.createElement('tr');

    // 2b. Criar cada célula <td> e preenchê-la
    const celulaId = document.createElement('td');
    celulaId.textContent = item.id;

    const celulaNome = document.createElement('td');
    celulaNome.textContent = item.name;

    // IMPORTANTE: Antes de acessar propriedades de item.data,
    // SEMPRE verificamos se item.data existe E se a propriedade existe.
    // Usamos o operador && ("e lógico"): a segunda parte só é avaliada
    // se a primeira for verdadeira. Isso evita o erro:
    // "Cannot read properties of null".
    // Nem todos os objetos da API possuem o campo data preenchido.

    const celulaCor = document.createElement('td');
    if (item.data && item.data.color) {
      celulaCor.textContent = item.data.color;
    } else {
      celulaCor.textContent = '—';
    }

    const celulaCapacidade = document.createElement('td');
    if (item.data && item.data.capacity) {
      celulaCapacidade.textContent = item.data.capacity;
    } else {
      celulaCapacidade.textContent = '—';
    }

    const celulaPreco = document.createElement('td');
    if (item.data && item.data.price) {
      // Formatar o preço em reais (R$) com duas casas decimais
      celulaPreco.textContent = item.data.price.toLocaleString('pt-BR', {
        style: 'currency',
        currency: 'BRL'
      });
    } else {
      celulaPreco.textContent = '—';
    }

    // 2c. Anexar as células à linha
    linha.appendChild(celulaId);
    linha.appendChild(celulaNome);
    linha.appendChild(celulaCor);
    linha.appendChild(celulaCapacidade);
    linha.appendChild(celulaPreco);

    // 2d. Anexar a linha ao corpo da tabela
    corpoTabela.appendChild(linha);
  }
}

Entendendo o código:

Trecho O que faz
corpoTabela.innerHTML = '' Limpa a tabela inteira antes de redesenhar. Assim evitamos linhas duplicadas.
document.createElement('tr') Cria um novo elemento HTML <tr> (linha de tabela) na memória, sem inseri-lo na página ainda.
document.createElement('td') Cria uma nova célula <td>.
.textContent = valor Define o texto visível dentro da célula.
if (item.data && item.data.color) Verificação em duas etapas: primeiro verifica se item.data existe, depois se a propriedade color existe. O operador && garante que a segunda parte só é avaliada se a primeira for verdadeira. Sem isso, acessar null.color causaria um erro fatal. Use sempre esse padrão ao acessar propriedades de objetos que podem ser null.
.toLocaleString('pt-BR', {...}) Formata o número como moeda brasileira (R$ 1.999,90). Recebe o idioma (pt-BR) e opções de formatação.
.appendChild(elemento) Insere o elemento filho dentro do elemento pai. A ordem dos appendChild define a ordem das colunas.

1.2 — Implementar a função listarDispositivos()

Agora vamos buscar os dados da API e alimentar o vetor. Substitua o conteúdo da função listarDispositivos():

async function listarDispositivos() {
  try {
    // 1. Fazer a requisição GET para a API
    //    (GET é o método padrão do fetch — não precisamos configurar nada)
    const respostaHTTP = await fetch(URL_API);

    // 2. Converter a resposta HTTP em um objeto JavaScript
    //    (a API retorna texto JSON, o .json() transforma em objeto/vetor)
    const dados = await respostaHTTP.json();

    // 3. Salvar os dados no vetor local
    dispositivos = dados;

    // 4. Redesenhar a tabela com os novos dados
    renderizar();

    // 5. Informar o usuário
    mostrarMensagem(dispositivos.length + ' dispositivos encontrados.', 'sucesso');

  } catch (erro) {
    mostrarMensagem('Erro ao listar: ' + erro.message, 'erro');
  }
}

Entendendo o código:

Trecho O que faz
await fetch(URL_API) Faz uma requisição GET para o endereço da API e espera a resposta chegar. A palavra await pausa a execução da função até a rede responder.
await respostaHTTP.json() Pega o corpo da resposta HTTP (que é texto JSON) e converte em objetos JavaScript que podemos manipular.
dispositivos = dados Substitui todo o vetor local pelo que a API retornou.
renderizar() Chama a função que acabamos de criar para redesenhar a tabela.

Sobre o try / catch — tratamento de erros:

Quando fazemos uma requisição pela internet, muita coisa pode dar errado: a rede pode cair, o servidor pode estar fora do ar, a URL pode estar errada. Se isso acontecer e não estivermos preparados, o JavaScript simplesmente para de executar e o usuário fica sem entender o que houve.

O try / catch é a estrutura que nos protege desses problemas. Funciona assim:

try {
    // "Tente executar este bloco de código."
    // Se TUDO der certo, o catch é ignorado.
}
catch (erro) {
    // "Se QUALQUER linha acima falhar, caia aqui."
    // A variável 'erro' contém uma descrição do problema.
}

Analogia: Imagine que você está fazendo uma ligação telefônica (a requisição fetch). O try é a tentativa de fazer a ligação. Se a pessoa atender (servidor responder), tudo flui normalmente. Mas se der sinal de ocupado, a ligação cair ou o número não existir, o catch entra em ação — ele é o seu "plano B" para lidar com a falha sem que o programa trave.

Na prática, dentro do catch, temos acesso ao objeto erro, que possui a propriedade .message com uma descrição do que deu errado. É esse texto que exibimos ao usuário.

💡 Experimento: Para ver o catch funcionando, alterem temporariamente a URL da API para algo inválido (ex: https://api.inexistente.dev/objects), cliquem em "Listar Todos" e observem a mensagem de erro vermelha. Depois, desfaçam a alteração para voltar ao normal.

Passo 2 — Buscar por ID (GET com ID)

O que vamos fazer: Buscar UM dispositivo específico pelo seu ID e preencher o formulário com os dados retornados, para possibilitar edição.

Substitua o conteúdo da função buscarPorId():

async function buscarPorId() {
  // 1. Pegar o valor que o usuário digitou no campo ID
  const id = campoId.value.trim();

  // 2. Validar: se o campo está vazio, avisar e parar
  if (!id) {
    mostrarMensagem('Digite um ID para buscar.', 'erro');
    return;   // "return" encerra a função aqui, nada abaixo executa
  }

  try {
    // 3. Fazer a requisição GET, agora com o ID na URL
    //    (Template Literal: o acento grave permite inserir variáveis com ${})
    const respostaHTTP = await fetch(`${URL_API}/${id}`);

    // 4. Verificar se a API encontrou o objeto
    if (!respostaHTTP.ok) {
      mostrarMensagem('Dispositivo não encontrado (ID: ' + id + ').', 'erro');
      return;
    }

    // 5. Converter a resposta em objeto JavaScript
    const item = await respostaHTTP.json();

    // 6. Preencher o formulário com os dados retornados
    //    (isso facilita a edição — o aluno vê os dados atuais e altera o que quiser)
    if (item.name) {
      campoNome.value = item.name;
    } else {
      campoNome.value = '';
    }

    if (item.data && item.data.color) {
      campoCor.value = item.data.color;
    } else {
      campoCor.value = '';
    }

    if (item.data && item.data.capacity) {
      campoCapacidade.value = item.data.capacity;
    } else {
      campoCapacidade.value = '';
    }

    if (item.data && item.data.price) {
      campoPreco.value = item.data.price;
    } else {
      campoPreco.value = '';
    }

    // 7. Atualizar o vetor local: colocar somente este item para visualização
    dispositivos = [item];
    renderizar();

    mostrarMensagem('Dispositivo "' + item.name + '" encontrado.', 'sucesso');

  } catch (erro) {
    mostrarMensagem('Erro ao buscar: ' + erro.message, 'erro');
  }
}

Entendendo o código:

Trecho O que faz
.trim() Remove espaços em branco antes e depois do texto digitado.
`${URL_API}/${id}` Monta a URL com o ID. Exemplo: https://api.restful-api.dev/objects/3. O acento grave (`) permite inserir variáveis com ${} (Template Literal).
respostaHTTP.ok Propriedade booleana: true se o status HTTP é 200–299 (sucesso), false se é 404, 500 etc.
return Para a execução da função imediatamente. Nenhum código abaixo do return é executado.
if (item.data && item.data.color) Verifica se data existe antes de acessar color. Evita erro "Cannot read properties of null".
dispositivos = [item] Coloca apenas o item buscado no vetor (os colchetes criam um vetor com um único elemento).

Passo 3 — Cadastrar um novo dispositivo (POST)

O que vamos fazer: Ler os dados digitados no formulário, montar um objeto JavaScript, enviar para a API via POST e adicionar o item retornado ao vetor local.

Substitua o conteúdo da função cadastrarDispositivo():

async function cadastrarDispositivo() {
  // 1. Ler os valores dos campos do formulário
  const nome = campoNome.value.trim();
  const cor = campoCor.value.trim();
  const capacidade = campoCapacidade.value.trim();
  const preco = campoPreco.value;

  // 2. Validação mínima: nome é obrigatório
  if (!nome) {
    mostrarMensagem('O nome do dispositivo é obrigatório.', 'erro');
    return;
  }

  // 3. Montar o objeto no formato que a API espera
  //    (consulte a documentação da API para saber o formato)
  // parseFloat() converte texto para número decimal.
  // Se o campo estiver vazio, parseFloat('') retorna NaN (Not a Number).
  // Usamos o operador || para substituir NaN por 0.
  const precoNumerico = parseFloat(preco) || 0;

  const novoDispositivo = {
    name: nome,
    data: {
      color: cor,
      capacity: capacidade,
      price: precoNumerico
    }
  };

  try {
    // 4. Fazer a requisição POST
    //    Diferente do GET, aqui precisamos configurar:
    //    - method: qual verbo HTTP usar
    //    - headers: avisar que estamos enviando JSON
    //    - body: os dados convertidos de objeto JS para texto JSON
    const respostaHTTP = await fetch(URL_API, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(novoDispositivo)
    });

    // 5. Verificar se a resposta foi bem-sucedida
    if (!respostaHTTP.ok) {
      mostrarMensagem('Erro ao cadastrar. A API retornou status ' + respostaHTTP.status + '.', 'erro');
      return;
    }

    // 6. Converter a resposta (a API retorna o objeto criado, agora com um ID)
    const itemCriado = await respostaHTTP.json();

    // 6. Adicionar o novo item ao vetor local
    dispositivos.push(itemCriado);

    // 7. Redesenhar a tabela para mostrar o novo item
    renderizar();

    // 8. Limpar o formulário e informar o usuário
    limparFormulario();
    mostrarMensagem('Dispositivo "' + itemCriado.name + '" cadastrado! ID: ' + itemCriado.id, 'sucesso');

  } catch (erro) {
    mostrarMensagem('Erro ao cadastrar: ' + erro.message, 'erro');
  }
}

Entendendo o código — as partes novas:

Trecho O que faz

Retomando a manipulação do DOM: Observe que no Passo 1 já usamos document.createElement() e .appendChild() para criar elementos na página dinamicamente, e document.getElementById() para buscar elementos existentes. Essas são as duas operações fundamentais de manipulação do DOM:

  • Consultar (query): getElementById(), querySelector(), querySelectorAll() — encontrar elementos que já existem na página;
  • Criar: createElement(), .textContent, .appendChild() — fabricar elementos novos e inseri-los na página.

Toda a integração frontend–API gira em torno disso: buscamos os dados via fetch(), salvamos no vetor e então usamos essas operações do DOM para exibir os dados na tela.

Passo 6 (Bônus) — Clicar na linha para preencher o formulário

🎁 Este passo não é obrigatório, mas resolve um problema prático: copiar IDs longos da tabela é chato! Ao clicar numa linha, o formulário será preenchido automaticamente.

O que vamos fazer: Adicionar um addEventListener('click', ...) em cada <tr> criada pela função renderizar(), para que ao clicar na linha, os dados daquele dispositivo preencham o formulário.

Adicionem o trecho abaixo dentro da função renderizar(), logo antes do corpoTabela.appendChild(linha) (ou seja, antes de inserir a linha na tabela):

    // 2e. (BÔNUS) Ao clicar na linha, preencher o formulário com os dados
    linha.style.cursor = 'pointer';  // muda o cursor para indicar que é clicável
    linha.addEventListener('click', function() {
      campoId.value = item.id;
      campoNome.value = item.name || '';
      campoCor.value = (item.data && item.data.color) ? item.data.color : '';
      campoCapacidade.value = (item.data && item.data.capacity) ? item.data.capacity : '';
      campoPreco.value = (item.data && item.data.price) ? item.data.price : '';
    });

Entendendo o código:

Trecho O que faz
linha.style.cursor = 'pointer' Muda o cursor do mouse para a "mãozinha" ao passar sobre a linha, indicando visualmente que ela é clicável.
linha.addEventListener('click', function() {...}) Registra uma função que será executada quando o usuário clicar naquela linha específica.
(item.data && item.data.color) ? item.data.color : '' Operador ternário: se item.data e item.data.color existirem, usa o valor; senão, usa string vazia. É a versão compacta do if/else.

Perguntas Frequentes

"Cadastrei um dispositivo, mas ao recarregar a página ele sumiu!" Isso é normal. O vetor dispositivos fica na memória do navegador (RAM). Ao recarregar a página, o JavaScript reinicia do zero e o vetor começa vazio. Além disso, a API de teste não persiste dados — ela é reiniciada periodicamente. Em aplicações reais, os dados ficam salvos num banco de dados no servidor.

"Busquei por ID e agora só vejo 1 dispositivo na tabela. Cadê os outros?" Ao buscar por ID, substituímos o vetor inteiro por dispositivos = [item] (um vetor com só aquele item). Para ver todos novamente, clique em "Listar Todos".

"O preço apareceu como NaN na tabela." Isso acontece quando o campo preço está vazio ou contém texto inválido. A função parseFloat('') retorna NaN. Por isso usamos parseFloat(preco) || 0 — o operador || substitui NaN por 0.
